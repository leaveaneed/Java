Если в блоках case отсутствует ключевое слово break, выполнение будет "проваливаться" (fall-through) в следующий блок case. Это значит, что после первого совпадения значение продолжит сравниваться и выполняться все последующие блоки до break или default.
int value = 2;
int i = 0;

switch (value) {
    case 1:
        i = 1;
        break;
    case 2:
        i = 2; // нет break!
    case 3:
        i = 3;
        break;
}

System.out.println(i);

Если value = 2, то код "провалится" в case 3, так как отсутствует break после case 2.
Вывод в консоль: 3.
